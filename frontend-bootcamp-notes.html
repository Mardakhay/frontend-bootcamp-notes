ğŸŒWhat Happens When You Type a URL

This is the most important mental model for any frontend developer. Everything the browser does is a consequence of this chain:

User types: https://example.com/users 1. DNS Lookup Browser â†’ DNS server "What is the IP for example.com?" DNS returns: 93.184.216.34 2. TCP 3-Way Handshake Client â†’ SYN â†’ Server Client â† SYN-ACK â† Server Client â†’ ACK â†’ Server Connection established. 3. TLS Handshake (HTTPS only) Exchange certificates, agree on encryption â†’ Secure channel created 4. HTTP Request GET /users HTTP/1.1 Host: example.com Accept: text/html 5. Why this matters When network errors happen in your React app, you'll read the Network tab and immediately understand what went wrong: DNS failures show as no response at all; HTTP errors like 401 Unauthorized, 404 Not Found, and 500 Server Error show as failed requests with a status code.

ğŸ¯Specificity â€” Why Your CSS "Doesn't Work"

When multiple rules target the same element, specificity determines which one wins. This is the source of most "why isn't my CSS applying?" bugs.

Specificity Score (calculated as a 3-part number): [ID selectors, Class/attr selectors, Element selectors] h1 â†’ 0-0-1 .title â†’ 0-1-0 #main â†’ 1-0-0 div.card â†’ 0-1-1 #app .card h2 â†’ 1-1-1 style="" â†’ wins over all (inline) !important â†’ nuclear option, beats even inline styles Higher number = wins. Compare left-to-right. 0-1-0 beats 0-0-100 (yes, 100 elements can't beat 1 class)
Professional Rule Never use !important except as a last resort. It destroys the cascade and makes code unmaintainable. If you need it, your specificity architecture is broken.

Flexbox, CSS Grid & Responsive Design

â†”ï¸Flexbox â€” Mental Model

Flexbox is a 1-dimensional layout system. Think of it as: "arrange items along one axis". Perfect for navigation bars, button groups, card rows.

Main axis (default: horizontal â†’): â”Œâ”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â” â”‚ [Item A] [Item B] [Item C] â”‚ â”‚ â”‚ â””â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”˜ Cross axis (perpendicular â†•): Controls vertical alignment of items
.container {
  display: flex;

  /* Main axis distribution */
  justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;

  /* Cross axis alignment */
  align-items: stretch | flex-start | flex-end | center | baseline;

  /* Direction */
  flex-direction: row | row-reverse | column | column-reverse;

  /* Allow wrapping */
  flex-wrap: nowrap | wrap;

  /* Spacing between items */
  gap: 16px;
}

/* Child control */
.item {
  flex: 1;           /* grow to fill available space */
  flex: 0 0 200px;   /* fixed 200px, don't grow/shrink */
  flex: 1 1 auto;   /* grow AND shrink from natural size */
}

âŠCSS Grid â€” Mental Model

Grid is a 2-dimensional layout system. Rows AND columns simultaneously. Use for full-page layouts, dashboards, image galleries.

.layout {
  display: grid;

  /* Define columns */
  grid-template-columns: 1fr 1fr 1fr;         /* 3 equal columns */
  grid-template-columns: repeat(3, 1fr);       /* same */
  grid-template-columns: 200px 1fr 200px;     /* sidebar | main | sidebar */
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));  /* responsive auto-cols */

  /* Define rows */
  grid-template-rows: auto 1fr auto;   /* header | main | footer */

  gap: 24px;
}

/* Spanning cells */
.hero {
  grid-column: 1 / -1;    /* span ALL columns */
  grid-column: 1 / 3;     /* span column 1 and 2 */
}
Grid vs Flexbox â€” Quick Rule Grid: you know the layout structure first, then place items in it. Flexbox: items determine the layout. Grid for pages, Flexbox for components.

ğŸ“±Responsive Design â€” Mobile-First Approach

/* MOBILE FIRST: start with mobile styles (no media query) */
.grid {
  display: grid;
  grid-template-columns: 1fr;    /* 1 column on mobile */
  gap: 16px;
}

/* Tablet: 768px and up */
@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Desktop: 1024px and up */
@media (min-width: 1024px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

/* Fully responsive grid (no media queries needed!) */
.auto-grid {
  grid-template-columns: repeat(auto-fill, minmax(240px, 1fr));
}

Why mobile-first? Forced to prioritize essential content. Easier to add complexity as screen grows. Better performance on slow mobile connections. Current industry standard.

âš¡Arrow Functions vs Regular Functions â€” The Key Difference

// Function declaration â€” hoisted, has own 'this'

React Hooks & Component Composition

ğŸªuseEffect â€” Side Effects in React

useEffect runs code after the component renders. Use it for: fetching data, subscriptions, timers, DOM manipulation.

import { useState, useEffect } from 'react';

function UserProfile({ userId }: { userId: number }) {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    // Run after every render where userId changed
    let cancelled = false;  // prevent race conditions

    async function load() {
      try {
        setLoading(true);
        const data = await fetchUser(userId);
        if (!cancelled) setUser(data);
      } catch (e) {
        if (!cancelled) setError('Failed to load user');
      } finally {
        if (!cancelled) setLoading(false);
      }
    }

    load();

    // Cleanup function â€” runs before next effect or unmount
    return () => { cancelled = true; };

  }, [userId]);  // â† dependency array: re-run when userId changes
  //   []        â€” run once on mount only
  //   no array  â€” run on every render (usually a bug)

  if (loading) return <p>Loading...</p>;
  if (error) return <p>{error}</p>;
  return <p>{user?.name}</p>;
}

Race condition: multiple fetches are in-flight, and the first one might resolve after the second. Fix: use a cancelled flag in the cleanup function (as shown above), or use AbortController: const ac = new AbortController(); fetch(url, {signal: ac.signal}); return () => ac.abort();

// useUsers.ts â€” reusable data fetching hook
interface UseUsersReturn {
  users: User[];
  loading: boolean;
  error: string | null;
  refetch: () => void;
}

export function useUsers(): UseUsersReturn {
  const [users, setUsers] = useState<User[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);
  const [trigger, setTrigger] = useState(0);

  useEffect(() => {
    let cancelled = false;
    setLoading(true);
    setError(null);

    usersService.getAll()
      .then(data => { if (!cancelled) setUsers(data); })
      .catch(e => { if (!cancelled) setError(e.message); })
      .finally(() => { if (!cancelled) setLoading(false); });

    return () => { cancelled = true; };
  }, [trigger]);

  return { users, loading, error, refetch: () => setTrigger(t => t + 1) };
}

      setTasks(backup);  // rollback on error
    }
  };

  return { tasks, deleteTask };
}
